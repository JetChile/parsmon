# ParsMon → Telegram: монитор алертов

Бот следит за двумя закреплёнными сообщениями в группе и при изменении шлёт
алерт в чат алертов. **Алерты операторам отправляются на испанском.**
В алерт попадают:

- **счётчики** по пунктам **5** (error crítico / критическая ошибка),
  **8** (sin conexión / нет связи), **9** (fuera de zona / за зоной
  завершения), **11** (alarma / тревога), **2.1** (batería baja >12h /
  НЗ >12ч) — из сообщения «Чек #N»;
- **номера самокатов** (S.xxxx) по этим же категориям — из второго сообщения;
- при появлении **нового чека** — отметка «Verifique el estado en la
  aplicación técnica» (проверьте статус в техническом приложении).

Алерт отправляется **только при изменении** (новый чек, изменился счётчик или
появились новые номера). Новые номера помечаются 🆕.

Зависимостей нет — только стандартный Python 3 (urllib).

> Исходные сообщения на русском, поэтому правила распознавания внутри
> `monitor.py` остаются на русском. Текст, который видят операторы в алертах, —
> на испанском.

## Что нужно один раз настроить в Telegram

1. Бот **@parsmonbot** должен состоять в обеих группах: источник
   (`-1002300878026`) и чат алертов (`-1003727768555`).
2. Для режима `getupdates` бот должен **видеть сообщения группы**. Варианты:
   - в @BotFather: `/setprivacy` → выбрать @parsmonbot → **Disable**; **или**
   - сделать бота **админом** группы-источника.
3. Проверить точные id чатов: добавьте бота, отправьте любое сообщение и
   откройте `https://api.telegram.org/bot<TOKEN>/getUpdates` — в ответе будет
   `chat.id`. Id супергруппы выглядит как `-100…`.

> ⚠️ Важно: режим `getupdates` «забирает» апдейты бота. Если этот же токен уже
> где-то опрашивает Telegram (другой процесс с getUpdates или webhook), будет
> конфликт. В этом случае используйте `READ_MODE=forward` (читает оба
> сообщения по message_id, не трогая очередь апдейтов) — см. ниже.

## Установка на сервер (systemd)

```bash
sudo mkdir -p /opt/parsmon
sudo cp monitor.py /opt/parsmon/
sudo cp .env.example /opt/parsmon/.env
sudo nano /opt/parsmon/.env          # проверьте токен и id чатов

# проверка парсера без сети:
python3 /opt/parsmon/monitor.py --selftest

# установка сервиса:
sudo cp parsmon-monitor.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now parsmon-monitor

# логи:
journalctl -u parsmon-monitor -f
```

При первом запуске бот пришлёт стартовый снимок (текущие счётчики и самокаты),
дальше — только при изменениях.

### Обновление / перезапуск

```bash
sudo cp monitor.py /opt/parsmon/        # после правок кода
sudo systemctl restart parsmon-monitor
sudo systemctl status parsmon-monitor   # проверить, что Active: running
```

### Сброс состояния (если нужно прислать снимок заново)

```bash
sudo systemctl stop parsmon-monitor
sudo rm -f /opt/parsmon/state.json
sudo systemctl start parsmon-monitor
```

## Режимы чтения (`READ_MODE`)

- **`getupdates`** (по умолчанию, для сервера): бот слушает все сообщения
  группы и сам распознаёт оба сообщения по содержимому («Чек #…» и список
  S.xxxx). Реагирует на правки почти мгновенно. Требует выключенный privacy
  mode или права админа.
- **`forward`**: читает ровно два сообщения по их `message_id`. В Telegram
  ПКМ по сообщению → «Копировать ссылку» → последнее число в ссылке это
  message_id. Заполните `CHEK_MSG_ID` и `SCOOTER_MSG_ID`. Не конфликтует с
  другими потребителями токена. Технически бот пересылает сообщение в
  `LOG_CHAT_ID`, читает текст и сразу удаляет пересылку.
- **`getchat`**: читает только последнее закреплённое сообщение (одно из двух).

## Запуск вручную / тест

```bash
python3 monitor.py --selftest   # парсинг встроенных примеров, без сети
python3 monitor.py --once       # один цикл опроса (нужны переменные окружения)
python3 monitor.py --loop       # непрерывно (то, что делает systemd)
```

## Как настроить пункты/категории и язык подписей

В начале `monitor.py` — словарь `POINTS`. Ключ → (подпись на испанском,
префикс строки в «Чек #N», подстрока заголовка во втором сообщении). Чтобы
изменить текст подписи — правьте первый элемент; чтобы добавить/убрать пункт —
правьте `POINTS` и список `ORDER`. Префиксы и подстроки должны совпадать с
русским текстом исходных сообщений. Тексты заголовков алертов («Nuevo
chequeo», «Verifique el estado…») — в функции `format_alert`.
